架构设计

expo-liteDBStore 的架构设计和核心组件

分层架构

API 层

对外暴露的接口，提供简洁易用的 API

  • 提供统一的 API 入口
  • 处理参数验证和类型转换
  • 返回标准化的结果和错误

核心层

业务逻辑处理，包括事务管理、查询引擎等

  • 事务管理
  • 查询引擎
  • 索引管理
  • 元数据管理

存储层

文件系统操作，处理数据的持久化存储

  • 文件读写操作
  • 分块存储管理
  • 存储适配器

工具层

通用工具函数，包括加密、错误处理等

  • 加密解密
  • 错误处理
  • 类型转换
  • 时间工具

核心组件

FileOperationManager

文件操作管理，处理文件的读写、分块、合并等操作

  • 支持单文件和分块文件的读写
  • 自动处理文件大小限制
  • 提供文件锁定机制，防止并发冲突
  • 支持文件压缩和加密

TransactionManager

事务管理，确保数据操作的原子性、一致性、隔离性和持久性

  • 支持 ACID 事务特性
  • 提供 beginTransaction、commit 和 rollback 方法
  • 使用日志机制确保事务的持久性
  • 支持嵌套事务

QueryEngine

查询引擎，处理复杂的查询条件和数据过滤

  • 支持多种查询操作符：$eq, $ne, $gt, $gte, $lt, $lte, $in, $nin, $and, $or
  • 支持排序、分页和投影
  • 自动优化查询计划
  • 支持索引扫描和全表扫描

IndexManager

索引管理，提高查询效率

  • 支持自动创建和手动创建索引
  • 支持单字段和复合索引
  • 自动维护索引的一致性
  • 支持索引优化和重建

MetadataManager

元数据管理，存储表结构、索引等元数据信息

  • 存储表的结构信息
  • 存储索引的定义和状态
  • 支持元数据的版本管理
  • 提供元数据的备份和恢复

CacheManager

缓存管理，提高数据访问速度

  • 多级缓存设计：内存缓存、磁盘缓存
  • 支持 LRU 缓存淘汰策略
  • 自动维护缓存的一致性
  • 支持缓存预热和预加载

KeyManager

密钥管理，处理数据加密和解密的密钥

  • 支持多种加密算法
  • 提供密钥生成和管理功能
  • 支持密钥轮换和更新
  • 保护密钥的安全存储

StorageAdapter

存储适配器，提供统一的存储接口，支持不同的存储后端

  • 提供统一的存储接口
  • 支持多种存储后端：文件系统、内存、SQLite 等
  • 支持存储后端的动态切换
  • 提供存储性能监控

数据持久化

从 API 调用到数据持久化的完整流程

  1. 应用调用 API 层的方法
  2. API 层验证参数并调用核心层组件
  3. 核心层处理业务逻辑，如查询、更新等
  4. 核心层调用存储层进行数据读写
  5. 存储层处理文件操作，包括分块、加密等
  6. 数据持久化到磁盘
  7. 返回结果给应用

性能优化

分块存储

单文件超过 5MB 自动分块，避免 RN FS 的限制

索引优化

自动创建和维护索引，提高查询效率

缓存机制

多级缓存设计，减少磁盘 I/O

批量操作

支持批量写入和更新，减少磁盘操作次数

异步操作

所有操作都是异步的，不阻塞主线程

安全特性

加密存储

支持 AES-GCM 加密，密钥由用户完全控制

数据完整性

使用校验和确保数据完整性

权限控制

支持基本的权限控制，保护敏感数据

可扩展性

模块化设计，支持自定义存储适配器、加密算法等

自定义适配器

可以实现自定义的存储适配器，支持不同的存储后端

自定义加密

可以实现自定义的加密算法，满足特定的安全需求

自定义查询

可以扩展查询引擎，支持更复杂的查询条件
