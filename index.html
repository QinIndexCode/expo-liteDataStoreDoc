expo-liteDBStore

为 React Native + Expo 项目设计，无需任何 native 依赖，提供完整的本地存储解决方案。

开始使用 API 参考

核心特性

零依赖、零配置

仅依赖 React Native FS，无需 Metro 配置，开箱即用。

完整 TypeScript 支持

完整类型定义，开发即使用，提供优秀的开发体验。

支持加密存储

可选择 AES-GCM 加密，密钥由您完全控制，保护敏感数据。

支持事务操作

完整的事务支持，确保数据一致性，避免数据损坏。

支持复杂查询

支持 where、skip、limit 等查询条件，满足复杂业务需求。

支持分块存储

单文件最大 5MB 自动分块，完美适配 RN FS 限制，支持大量数据存储。

快速开始

安装 

npm install expo-lite-db-store

或者使用 yarn 或 pnpm 安装

基本使用 

import { createTable, insert, findOne } from 'expo-lite-db-store';

// 创建表
await createTable('users');

// 插入数据
await insert('users', [{ id: 1, name: '张三', age: 25 }]);

// 查询数据
const user = await findOne('users', { id: 1 });
console.log(user); // { id: 1, name: '张三', age: 25 }

架构设计

分层设计

  • API 层： 对外暴露的接口，提供简洁易用的 API
  • 核心层： 业务逻辑处理，包括事务管理、查询引擎等
  • 存储层： 文件系统操作，处理数据的持久化存储
  • 工具层： 通用工具函数，提供加密、压缩等功能

核心组件

  • FileOperationManager： 文件操作管理，处理文件的读写和管理
  • TransactionManager： 事务管理，确保数据操作的原子性
  • QueryEngine： 查询引擎，处理数据的查询和过滤
  • IndexManager： 索引管理，提高查询性能
  • MetadataManager： 元数据管理，存储表结构和配置信息
